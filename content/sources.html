Our sources
(making sure we only make requests when necessary)

When someone looks for Open Access research using the Button, we want to be sure the article or data is not already available before we contact authors. We can't search the entire web, so we rely on up-to-date indexes of institutional and thematic repositories from around the world.

Starting with identifiers

We start off by pulling identifying information about the research, which in loose order of preference can be: URL, DOI, Pubmed ID, Pubmed Central ID, article title, or citation.

If given a URL, we check the URL itself to see if it contains a DOI, PMID, or PMC ID. If so, we extract that identifier. For example, here is a URL with a DOI in it:

http://jcheminf.springeropen.com/articles/10.1186/1758-2946-3-47

If we don't find an identifier in the URL, we search the page content for DOI, title, PMID, and PMC ID - if we find one, we use it as our priority identifier.

If given a full citation or a title, we search crossref for a DOI and title that appear to match. If we can't find anything from a citation, we try to pull the title, then check crossref again with just that.

Finding by Pubmed ID and Pubmed Central ID

Now, if we have a PMID or a PMC ID, we search Europe PMC. If we find an Open Access article there, we're done! If we find a paywalled article there, we try to find the DOI and title for it.

Finding by DOI

We prioritise DOI searches over title searches, and we search for DOIs in the following order, stopping as soon as we find a link to an open article:

  • Europe PMC
  • Share
  • OA DOI
  • Dissemin
  • BASE
  • Core
  • Openaire
  • Figshare
  • DOAJ

Do you think we should add more sources? Let us know!

If we don't find the research we're looking for, we try to grab other information, such as the title. If that doesn't work, we check crossref with the DOI and extract a title if available.

Finding by title

If we have a title, we go back to the sources where title searches are accepted (sometimes records in these sources don't include the DOI):

  • Europe PMC
  • Share
  • BASE
  • Openaire
  • Core
  • Figshare
  • DOAJ

Last resort

If we can't find the article anywhere, but we have the journal name, we check DOAJ to see if the journal is registered as Open. If it is, we return the URL to the journal itself, so that the user can look for the article manually.

What about "open but needs login"?

Unfortunately, many repositores contain "open" articles, but require a login - which is not very open. Wwe manage a list of such repositories, and, if a URL matches one, we do an additional check to see if the item can be accessed without a login. If it can't, we don't consider it open.

(There are a few other sites that we blacklist outright, for various reasons.)

What about institutional access?

Some of our users are members of institutions, such as universities, that have additional access rights to some content. In these cases, we'd love to collaborate on customised plugins, bookmarklets, and search methods (such as integration with your library ex-libris search) that check whether or not "local" access is possible. You can also build your own using our API.

We also hook up to inter-library loan systems and send alerts or triggers for ILL requests! If you're interested in this, contact us.

If all else fails, start a request

When we can't find an open version of research anywhere, it's time to start a request! We search the page content and metadata for the principal author's name and email address, and create a new request on our system. We collect accessibility and licence information from DOAJ, Sherpa Romeo, and Lantern. Then we contact the author and ask them to provide a legally accessible copy - either with a URL to something we couldn't find, or by uploading it. If an author uploads an article to us, we store it in Zenodo, so it's available long-term. Where the request is for a research dataset, we store uploads with the Open Science Foundation.

Our user community also help to fulfill requests, by providing author email addresses, additional metadata, or additional sources. Users can also add their support to others' requests. By collecting the stories of users requesting access, we present compelling arguments why an author should share their research.
